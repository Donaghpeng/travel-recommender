# 目的地扩增规划书

> 版本：v1.0（最终定稿）
> 目标：当前 58 个目的地 → 300~500 个国内目的地

---

## 一、现状与目标

**现状**：58 个目的地，每种偏好类型只有个位数候选 → 推荐结果重复率居高不下。

**目标**：扩增到 300~500 个国内目的地，每个"地域 × 类型"格子至少 5~10 个选项，让每次推荐都能翻出"新面孔"。

**已确认方向**：
- 只专注国内 ✅
- 稳定为主，偶尔新面孔（每次结果至少 2 个新的）✅
- 高德 POI API 数据源 ✅（Key 已提供）

## 二、数据获取方案

### 核心数据：高德地图 POI API

```
高德文本搜索 POI → 获取景点名称、坐标、类型、评分、地址
```

- 调用方式：`https://restapi.amap.com/v3/place/text?key=xxx&keywords=景点&city=杭州&offset=100`
- 按城市逐个获取，每个城市 1~3 类 POI（风景名胜、人文古迹等）
- 去重：同名景点合并
- 预计调用量：100 个城市 × 平均 5 类 = 500 次请求（免费用量足够）

**第一轮测试范围（先跑小批）**：
- 江浙沪（江苏、浙江、上海）+ 广东 + 四川
- 约 5 省 × 10~20 城市 × 5~10 个 POI/城市 ≈ 100~200 个目的地数据
- 运行后给你看效果，确认后再跑全量

### 描述数据：DeepSeek AI 批量生成

> 方案说明：利用已配置的 `DEEPSEEK_API_KEY`，将获取到的目的地信息分批发给 DeepSeek，每批 20~30 个，让它为每个目的地生成 1~2 句中文描述。

**流程**：
1. 高德 POI 数据收集完成后 → 生成 `pois_raw.json`
2. 将目的地分批（每批 20~30 个描述请求）
3. 调用 DeepSeek API，Prompt 示例：
   ```
   请为以下旅游目的地各写 1~2 句中文简介（内容包括：景点特色、适合什么样的人去玩）。
   用 JSON 格式返回，键为目的地名称，值为简介内容。
   
   1. 莫干山 - 浙江湖州 - 自然风光
   2. 南浔古镇 - 浙江湖州 - 人文古迹
   ...
   ```
4. 解析返回结果，合并到目的地数据中
5. 失败的个别条目用模板描述兜底

**特点**：
- ⚡ 速度快：20 次 API 调用，2~3 分钟完成 500 个目的地
- ⭐ 质量好：DeepSeek 生成的描述比模板自然通顺，包含实用信息
- ❌ 不依赖外网：国内可直连 DeepSeek API
- 🏗️ 一次性脚本：只在构建阶段运行，不会触发搜索路径

## 三、人工复核标准

### 方式：抽样检查（约 20%）

**取样规则**：
1. **按类型抽样**：每种类型随机抽 20%
2. **按异常值抽样**：评分 < 2.0 或描述 < 20 字的自动标记
3. **按地区抽样**：每省至少抽查 1~2 个

### 核查项

| 项目 | 标准 | 不合格处理 |
|------|------|-----------|
| 名称 | 能看懂，无乱码 | ✅ 自动修正 |
| 描述 | 含位置、特色、适合人群 | ❌ 标记重跑 |
| 类型映射 | POI 分类 → 内部标签准确 | ❌ 手动修正 |
| 坐标 | 不偏离城市范围 | ✅ 自动验证 |
| 关键词 | 至少 2~3 个偏好标签 | ❌ 补关键词 |

## 四、推荐多样性控制

### 策略：「稳定但偶尔有新面孔」

**具体实现**：
```
候选池分层:
├── ① 核心池（80% 权重）：评分高、匹配度高的常规推荐
├── ② 探索池（20% 权重）：评分中等但匹配度尚可的小众目的地
└── ③ 新鲜池（与②部分重叠）：最近未出现过/新增的目的地
```

**每次展示规则**：
- 返回 10 个结果中，**至少 2 个来自探索池/新鲜池**
- 核心池内采用 **MMR 多样性排名**（同类型不连出超过 2 个）
- 探索池每 **7 天轮换一次** 候选列表
- 新增加的目的地默认进入探索池，积累一定"曝光"后升入核心池
