## 五、POI ↔ 推荐打通设计（NJ 指定的关键交互）

### 交互流

```
场景：用户在景点搜索中输入"西湖"

[POI 搜索结果]
┌──────────────────────────────┐
│ 🏛️ 西湖                  4.8⭐ │
│ 📍 杭州市 · 🏞️ 自然景观      │
│ 西湖是中国十大风景名胜之一... │
│                              │
│ [🎯 查看杭州推荐方案] ← 新增！ │
└──────────────────────────────┘

点击 [🎯 查看杭州推荐方案] →
→ 自动跳转到「目的地推荐」Tab
→ 杭州排在推荐列表顶部，展开详情
→ 同时高亮标记"你从「西湖」来到这里"
```

### 技术实现方式

在后端 `/api/poi-search` 返回结果中，每条 POI 已有 `city` 字段。前端只需：

1. POI 卡片底部增加「查看该城市推荐方案」按钮
2. 点击后触发主搜索：用该城市作为偏好输入 `/api/recommend`
3. 或更简单：直接跳转到推荐 Tab 并自动选中目标城市

**最优方案（无需后端改动）：**
- POI 卡片底部按钮 → 调用 `/api/recommend?preferences=该城市`（或其他推荐参数）
- 推荐结果渲染时，在目标城市卡片加一个特殊标记

## 八、移除 / 降级的功能处理

### 移除

| 功能 | 移除理由 | 替代方案 |
|---|---|---|
| 美团集成 | 1-2 分钟等待对演示不友好，数据不稳定 | 无 |
| 导出 PDF | 演示场景不需要 | 无 |
| Unsplash 照片 | 当前用渐变背景，效果已够 | 无 |

### 降级

| 功能 | 降级方式 |
|---|---|
| 对比分析 | 仅保留结果预览卡右上角的小对比勾选框，去掉独立弹窗 |
| 日行程规划 | 折叠进卡片详情抽屉的「展开行程」 |
| 预订链接 | 放在详情抽屉最底部 |
| 评价详情 | 折叠进卡片详情抽屉的「用户评价」 |
| 航班追踪/价格 | 折叠进卡片详情抽屉的「航班信息」 |

## 十一、开放问题（待 NJ 确认）

1. **预览卡尺寸**：一行 3 张还是在搜索底部显示不同布局？若屏幕较小是否要自适应 2 列？

3. **Tab 切换时是否保留搜索状态**：比如在「景点探索」搜了"西湖"，切到「目的地推荐」Tab 时，搜索表单是否自动填入相关的城市/偏好？  
4. **天气预警形式**：全局工具栏的角标数字，点击后展开详情列表还是弹窗？
